caminho fácil (e seguro) só pra ver — zero mudança

Ver HT (Hyper-Threading)

Get-CimInstance Win32_Processor | Select NumberOfCores, NumberOfLogicalProcessors


→ se LogicalProcessors > NumberOfCores, o HT está ligado.

Ver o que já está setado

Se estiver no CMD (Prompt de Comando)

Mais fácil (2 linhas, uma de cada vez):

reg query "HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management" /v FeatureSettingsOverride
reg query "HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management" /v FeatureSettingsOverrideMask


Tudo em 1 linha (executa as duas consultas):

reg query "HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management" /v FeatureSettingsOverride & reg query "HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management" /v FeatureSettingsOverrideMask


Aquele ^ que eu tinha colocado é só “quebra de linha” no CMD para ficar legível; não é obrigatório. Pode ignorar e usar as versões acima.


(se não existir, significa “padrão do Windows”.)

Mapa rápido dos valores (só pra você interpretar o que o scanner pede):

0x00000048 → HT ligado, sem BHI

0x00002048 → HT desligado, sem BHI

0x00800048 → HT ligado, com BHI

0x00802048 → HT desligado, com BHI



/--------------------------------------- Correção 01
Criar o primeiro valor (FeatureSettingsOverride)

Painel da direita (área vazia) → botão direito → New → DWORD (32-bit) Value.

Nomeie exatamente: FeatureSettingsOverride → Enter.

Dê duplo clique nele:

Em Base, escolha Hexadecimal e digite 00802048 (sem 0x).
(ou escolha Decimal e digite 8396872)

OK.

Criar o segundo valor (FeatureSettingsOverrideMask)

Botão direito (área vazia) → New → DWORD (32-bit) Value.

Nome: FeatureSettingsOverrideMask → Enter.

Duplo clique → Base: Decimal → Value data: 3. (Se preferir Hexadecimal: 00000003) → OK.
/---------------------------------------
